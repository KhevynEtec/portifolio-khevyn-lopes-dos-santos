# 📊 Sistema de Triagem e Validação de Dados (Fluxograma e Pseudocódigo)

## 📝 Descrição do Projeto
[cite_start]Este projeto apresenta a modelagem lógica de um sistema de triagem e validação de dados, estruturado através de diagramas de fluxo e representações em pseudocódigo[cite: 2, 19]. [cite_start]O objetivo principal é estabelecer um pipeline de decisão que valide entradas de usuários e aplique filtros baseados em parâmetros técnicos e demográficos[cite: 3, 23].

[cite_start]Desenvolvido como parte do meu portfólio técnico, o sistema processa variáveis específicas para direcionar o fluxo de atendimento ou processamento de dados[cite: 5, 22]. [cite_start]A lógica abrange desde a validação inicial do dado até a finalização do processo (FIM), garantindo que cada etapa siga critérios rigorosos de decisão[cite: 3, 17].

## 🚀 Tecnologias e Conceitos Utilizados
* [cite_start]**Modelagem de Processos:** Estruturação de fluxogramas para visualização de decisões lógicas[cite: 1, 4].
* [cite_start]**Lógica de Programação:** Implementação de estruturas condicionais (SIM/NÃO) e loops de repetição[cite: 9, 14, 24].
* [cite_start]**Algoritmos em Pseudocódigo:** Definição de passos computacionais para manipulação de variáveis[cite: 20, 25].
* [cite_start]**Parâmetros Técnicos:** Gerenciamento de variáveis como indicadores de pressão ou idade[cite: 22, 23].

## 📊 Estrutura da Lógica e Resultados
O projeto demonstra a eficácia na organização de fluxos complexos, tratando diferentes cenários de entrada:

* [cite_start]**Validação de Integridade:** O sistema inicia com a função "Valida dado", garantindo que a informação de entrada seja consistente[cite: 3].
* [cite_start]**Filtros Condicionais:** Aplicação de regras baseadas em valores como $VA-80$ e limites de controle como $Don=10$[cite: 5].
* [cite_start]**Segmentação de Dados:** Utilização de critérios como idade (ex: `Idade 65`) para ramificação do atendimento[cite: 23].
* [cite_start]**Estrutura de Repetição:** Presença de comandos de controle como `WH` (While) e encerramento de processos[cite: 17, 24].
